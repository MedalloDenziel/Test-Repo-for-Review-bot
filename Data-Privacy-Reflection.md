# Data-Privacy-Reflection - Denziel Medallo
## Reflection
### What steps can you take to ensure you handle data securely in your daily tasks?
* I will ensure that I never share internal GitHub links or code snippets containing configuration keys in public forums or Discord channels. I will also make it a habit to lock my laptop whenever I step away from my workspace and use a password manager to keep my development credentials secure. 
### How should you store, share, and dispose of sensitive information safely?
* Sensitive information should only be shared through approved company channels (like encrypted DMs or secure document shares) and never over personal social media. When disposing of sensitive digital information, I will ensure files are permanently deleted rather than just moved to the trash bin, and I will follow company protocols for clearing cache or local storage when testing features.
### What are some common mistakes that lead to data privacy issues, and how can they be avoided?
* A common mistake is hard-coding API keys or secrets directly into the source code, which can then be leaked if the repository becomes public. This can be avoided by using environment variables (`.env` files) and ensuring they are listed in the `.gitignore` file so they are never pushed to GitHub.
